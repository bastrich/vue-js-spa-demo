Report

Daniil Bastrich

Introduction

This is the site of imaginary (or not) conspiracy community who seeks for the world domination. Here you can find out about their purposes, their rules, meeting schedule and try to join them.

Inspiration

  1. Existing conspiracy theories like for example "New World Order" or "Illuminati".
  2. Action and adventure movies/TV series showing secret ancient communities (for example, The Old Guard (2022), The Umbrella Academy(2019)).
  3. Strange sites on the Internet with obscure content and unknown authors (see Resources section for examples).

Accessibility

  1. Desktop Web
  2. Desktop Web with browser accessibility features (like Text To Speech)
  3. Mobile Web

Usability

  1. It should have pleasant but not too calm color scheme in order to make the visitors feel a bit tense. Eventually this is a secret conspiracy society.
  2. Design in the whole should not be overloaded but at the same time should not be minimalistic especially considering a tense color scheme. Overloaded design with tense colors will cause a quite negative user experience.
  3. Navigation should be as simple and straightforward as possible.

Learning

  1. Modern CSS features like Flexbox and Grid Layout. I've been learning CSS about 10 years ago, and I was surprised now when met those new great and much more comfortable features than using float.
  2. Vuw JS - nice and useful JS-framework which allows to create Single Page Applications easily and provides a template engine. Never worked with it before, so now I am a bit familiar with that framework.
  3. Dived a bit deeper to semantic HTML layout. I thought that was a deprecated approach, but looks like it is quite important for moder Accessibility features like, for example, Text To Speech.
  4. Learned some basics in how to use Figma.

Evaluation I

As I already had technical experience in web development, many tasks were easier for me and took less time. Using and debugging CSS, integration of JS template library were smooth enough. And I didn't have any long blockers or problems I couldn't resolve.

Evaluation II

VueJS is generally recommended to use with npm package manager in the mode with precompiled JS. So there were some difficulties when trying to make it work right in the browser without any pre-compilation. That took some time and there are still some limitations, but generally I made it work fine for my purposes.
